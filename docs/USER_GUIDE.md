# Asiana Hub — User Guide

---

## Getting Started

### Registering an Account

1. Visit the Asiana Hub URL provided by your administrator
2. Click **Request access** on the login page
3. Fill in your full name, work email, and a password
   - Password must be at least 8 characters, include one uppercase letter and one number
4. Click **Request access**
5. Wait for your administrator to approve your account
6. You will receive an **email notification** once approved

### Logging In

1. Visit the Asiana Hub URL
2. Enter your email and password
3. Click **Sign in**

### Forgot Your Password?

1. Click **Forgot password?** on the login page
2. Enter your email address
3. Check your inbox for a reset link (expires in 1 hour)
4. Click the link and set a new password

## For Requesters

Requesters can submit money advance requests on behalf of students.

### Submitting a New Request

1. Click **New Request** in the left menu or on the dashboard
2. **Search for the student** — type their name or student ID in the search box
3. Select the student from the dropdown list
4. If the student does not exist yet:
   - Click **Add "[name]" as new student**
   - Enter their full name and student ID number
   - They will be saved permanently for future use
5. The **Advance Frequency** will automatically show how many times this student has requested before
6. Enter the **Amount** in RM (maximum RM200, whole numbers only)
7. Click **Submit Request**

### Viewing Your Requests

1. Click **My Requests** in the left menu
2. You can see all your submitted requests and their current status

### Understanding Request Statuses

| Status | Meaning |
|---|---|
| **Pending** | Submitted, waiting for an approver to review |
| **Completed** | Approved and paid to the student — request closed |

### Searching and Filtering

Use the filter bar at the top of **My Requests** to narrow down results:
- **Search box** — type a student name or ID
- **Status dropdown** — filter by request status
- **Date fields** — filter by submission date range
- Click **Clear filters** to reset all filters

## For Approvers

Approvers review pending requests and manage the advance lifecycle.

### Viewing All Requests

1. Click **All Requests** in the left menu
2. All requests from all requesters are visible here
3. Use the search and filter options to find specific requests

### Actioning a Request

1. Click **View** on any request in the list
2. Review the request details:
   - Student name and ID
   - Amount requested
   - Advance frequency (how many times this student has requested)

### Approving a Request (Disbursing the advance)

1. Open the request detail page
2. Click **Approve**
3. Optionally add a comment
4. Click **Confirm Approval**
5. The request moves straight to **Completed** status

> Approving means you have reviewed **and** paid the advance to the student. There is no separate repayment step — approval completes the request.

### Copying the Bank Transfer Reference

Before making the bank transfer:
1. On the request detail page, find the **Bank Transfer Reference** box
2. Click **Copy reference**
3. The reference text is copied to your clipboard — example:
   ```
   10062026 AHMAD BIN ALI JUN26 ADVANCE3 RM150
   ```
4. Paste this into the **reference/description** field when making the bank transfer

### Deleting a Pending Request

If a request needs to be cancelled before it is approved:
1. Open the request detail page (must be **Pending** status)
2. Click **Delete Request**
3. Optionally add a reason/comment
4. Confirm — the request is removed from normal views

> **Note:** Approvers can only delete **Pending** requests. Administrators can delete requests at any status.

## Dashboard

After logging in, the **Dashboard** shows:
- Quick stats: number of pending, completed, and total requests
- Feature cards — click **Money Advance** to go to the requests module

---

## Tips

- **Student names are stored in uppercase** — this is intentional for the bank reference format
- **Frequency is automatic** — you don't need to count; the system tracks it
- **The bank reference changes daily** — always copy it fresh on the day of transfer, not in advance
- **Completed requests are kept forever** — you can always search historical records
